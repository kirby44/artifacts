CyBench evaluation: work summary and effort

Overview

Between April 22 and May 19, 2026 we stood up the CyBench environment on Kubernetes, validated all 39 tasks, and ran 243 evaluations across 20 run configurations. The work falls into two phases: phase 1 (Apr 22 - May 10), environment setup plus baseline evaluations, and phase 2 (May 13-19), a study of how much the agent harness matters. Evaluation runs themselves mostly executed unattended overnight; the effort figures below count human working time, not eval wall-clock.

bars = sustained work bursts · dots = single-day documentation work (May 1, 8, 10; final chart polish May 23)

Day-by-day log

Phase 1: setup and baseline evaluations (Apr 22-30)

Apr 22 · Environment bring-up and smoke tests initial analysis · ~1 d

Stood up CyBench on Kubernetes (minikube) and smoke-tested all 39 tasks with a small model (46 short evals in two batches). Outcome: 38/39 environments viable; one task (chunky) cannot schedule on an 8 GiB cluster.

Apr 22-23 · Solution-script verification initial analysis · ~0.5 d

Ran the official solution scripts across all reachable tasks (37 evals) to confirm each environment actually solves. Found one more broken task under Kubernetes. Together with the smoke tests this fixed the working set at 35-36 scoreable tasks and a 4-task exclusion list.

Apr 23 · Baseline run: Claude Opus 4.7, easy variant initial evaluation · ~1 d

The headline baseline: 36 tasks, message limit 50, run serially. Result: 23/35 = 65% solve rate. About 2 hours of eval wall-clock.

Apr 23-24 · Retry rounds 2 and 3 initial evaluation · ~0.5 d

Best-of-N reruns on the failures (11 tasks in round 2, 5 in round 3) to distinguish stable failures from flaky ones.

Apr 27 · GPT comparison run additional evaluation · ~1 d

Full 36-task run with a GPT model plus a 9-task retry round and one hard-variant probe. Also wrote a manual walkthrough of the diffecient solution path, a task both models struggled with.

Apr 28 · Results analysis: refusals, gaps, reasoning config analysis · ~1 d

Built two refusal scanners (LLM-based and static) and ran them across the trajectories, wrote a content-filter breakdown, completed run reports and a gap analysis for both the Claude and GPT runs, and investigated reasoning-effort API options to prepare the extended-reasoning experiment.

Apr 28-30 · Extended-reasoning (xhigh) run and failure taxonomy additional evaluation, analysis · ~2 d

35 tasks at maximum reasoning effort over three evenings, plus retry rounds. Verdict: 22/35 = 63%, within noise of baseline; xhigh flipped 2 fails to passes but 3 passes to fails. Closed with a per-task failure-mode taxonomy of the 7 persistent failures (conceptually stuck, budget burned on environment issues, silent stops).

Phase 1, continued: documentation (May 1-10)

May 1 / 8 / 10 · Documentation and feasibility notes reporting, analysis · ~1.5 d

Scaffold documentation (May 1), environment and tech-stack documentation with all reports organized into one place (May 8), and a feasibility study of applying a published agent-failure taxonomy (AGENTRX) to CyBench trajectories (May 10).

Phase 2: agent-harness study (May 13-19)

May 13 · Environment documentation to shareable HTML reporting · ~0.5 d

Converted the environment documentation into a self-contained HTML explainer for circulation.

May 13-14 · Claude Code harness integration and full run additional evaluation, analysis · ~1.5 d

Integrated the Claude Code agent harness into the evaluation framework and ran the full easy set with it. Same push produced deep-dives into the locktalk task (thinking vs no-thinking, success walkthrough, behavior when web access is blocked) and analyses of token-limit terminations.

May 15-16 · Harness ablations additional evaluation, analysis · ~1.5 d

Surveyed ~24 references on harness engineering and token overhead, then ran five ablations: stripping the skill list, stripping skills plus system prompt, a "diet" minimal-prompt variant, diet plus compaction, and diet across the full easy set. Followed up with per-task comparisons (diet vs baseline, ReAct vs diet) and a compaction-issues analysis.

May 17 · Native Claude Code comparison additional evaluation · ~0.5 d

Ran Claude Code natively (outside the evaluation framework) against CyBench tasks in docker. Smoke test passed; captured the default system prompt and analyzed token consumption of a successful run for comparison against the framework-mediated harness.

May 17-18 · GPT rerun with 5M-token budget additional evaluation, analysis · ~1 d

Full GPT rerun under a 5M output-token limit with best-of-N aggregation (any success counts; safety-filter failures tracked separately). Produced the never-solved-tasks analysis and a Claude Code vs naive-API comparison.

May 19 (+23) · Final harness report reporting · ~1 d

Final write-up of the harness study: failure modes, token-category decomposition figures, and cost-comparison charts (cost differences are smaller than raw token-count differences suggest). Light chart polish on May 23.

Deliverables

  • Validated CyBench environment on Kubernetes with a documented 4-task exclusion list and solution-script verification for every task.
  • Baseline report: Claude Opus 4.7, easy variant, 65% solve rate, with per-task results and gap analysis against retry rounds.
  • GPT comparison report (initial run plus 5M-token-budget rerun with best-of-N aggregation).
  • Extended-reasoning (xhigh) ablation report: no significant gain over baseline.
  • Refusal and content-filter analysis with reusable scanners.
  • Per-task failure-mode taxonomy for persistent failures.
  • Agent-harness study: Claude Code integration, five prompt/skill ablations, native-harness comparison, and token/cost decomposition figures.
  • Environment, scaffold, and tech-stack documentation in shareable form.
